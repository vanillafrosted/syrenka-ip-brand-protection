# Ray's World Library

**Purpose**: The permanent database of Ray's authentic world.

This document stores verified facts about Ray's real life, habits, locations, and routines in Japan. It supports editorial work across all platforms: articles, photography, video, social media, and future publications.

This is a knowledge database, not editorial interpretation. Each entry stores facts. Editors apply these facts according to their article's needs.

---

## How to Use This Library

Before writing or creating content:

1. Search the library by category
2. Find authentic details relevant to your article/project
3. Use only details that naturally fit your narrative
4. Do not force details into the story
5. Never invent details not in this database

Adding to the library:

When you discover a new authentic detail about Ray's life, add it to this database with full information. The library grows with each article.

## Markdown Documentation vs. Permanent Database

This markdown document serves as **editorial documentation**.

It explains the structure and contains starter entries for reference.

However, the **official Ray's World Library will ultimately be maintained as a spreadsheet or database**.

Why the distinction:

- **Markdown** = Documentation (easy to read, version-controlled in Git)
- **Spreadsheet/Database** = Permanent data source (searchable, sortable, updatable, scalable)

As the library grows beyond 50-100 entries, a structured database becomes more practical than markdown.

The spreadsheet will be the single source of truth for all Ray's authentic world facts.

The markdown documentation will reflect the spreadsheet structure for team reference.
